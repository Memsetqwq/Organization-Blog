## 一、项目概述

### 1.1 项目背景

本科成开放原子开源社团官网是社团对外展示的官方门户，承担以下核心职能：

- **形象展示**：展示社团概况、技术方向、核心成员、荣誉成果
- **信息传播**：发布社团动态、新闻、活动通知
- **人才招募**：提供加入申请入口，收集候选人信息
- **资源导航**：整合开源技术相关链接

### 1.2 项目基本信息

| 属性 | 说明 |
|------|------|
| **项目名称** | 科成开放原子开源社团官网 |

## 二、核心技术选型

### 2.1 为什么选择「零框架」纯前端方案？

**评审可能疑问**：市面上有 Vue/React 等成熟框架，为什么选择一个「纯原生 JavaScript + CSS」的项目？

**我们的回答**：

| 考量维度 | 传统框架（如 React/Vue） | 本项目方案 |
|---------|-------------------------|-----------|
| **项目复杂度** | 适合中大型 SPA 应用 | 网站仅7个页面，路由简单 |
| **首屏加载** | 需要加载框架本体（~30KB+） | 零框架开销 |
| **维护成本** | 需要理解组件化、状态管理等概念 | 原生 DOM 操作，团队已熟悉 |
| **依赖管理** | npm 包版本更新频繁 | 仅有 Parcel 构建工具依赖 |
| **学习曲线** | 新成员需学习框架概念 | 有 JavaScript 基础即可上手 |

**结论**：本项目是一个**内容展示型网站**，而非需要复杂交互的 Web 应用。选用纯原生技术栈，既满足功能需求，又最大化开发效率与性能。

### 2.2 构建工具 - Parcel 2.x

#### 2.2.1 什么是 Parcel？

Parcel 是一款**零配置的打包工具**。对比传统打包工具（如 Webpack）需要编写大量配置文件，Parcel  автоматически 解析 HTML/CSS/JS 之间的依赖关系，直接输出优化后的构建产物。

#### 2.2.2 为什么选择 Parcel？

| 特性 | 说明 |
|------|------|
| **零配置** | 无需任何 `parcel.config.js` 等配置文件 |
| **开箱即用** | 安装后直接 `parcel src/index.html` 即可运行 |
| **自动依赖** | 自动扫描 HTML 中 `<link>` 和 `<script>` 标签的引用 |
| **热更新** | 开发模式文件修改后自动刷新浏览器 |
| **多核并行** | 使用 worker 进程并行编译，构建速度快 |

#### 2.2.3 实际效果

```bash
# 开发时，一行命令启动热重载服务器
npm run dev    # parcel src/index.html

# 部署前，一行命令生成优化产物
npm run build  # parcel build src/index.html
```

构建产物输出至 `dist/` 目录，可直接部署至 GitHub Pages、Vercel、Netlify 等静态托管服务。

### 2.3 样式系统 - CSS 变量 + 主题切换

#### 2.3.1 技术原理

**CSS 变量**（CSS Custom Properties）是 CSS3 引入的特性，允许在 `:root` 中定义可复用的值：

```css
:root {
  --bg: #0D1117;           /* 背景色 */
  --text: #E6EDF3;         /* 文字色 */
  --accent: #58A6FF;       /* 强调色 */
}
body {
  background-color: var(--bg);  /* 使用变量 */
  color: var(--text);
}
```

**主题切换**通过 `data-theme` 属性实现：

```html
<html data-theme="dark">  <!-- 或 data-theme="light" -->
```

```css
[data-theme="dark"] {
  --bg: #0D1117;
  --text: #E6EDF3;
}
[data-theme="light"] {
  --bg: #FFFFFF;
  --text: #1F2328;
}
```

#### 2.3.2 实现方案

主题管理由 `theme.js` 中的 `ThemeManager` 类统一负责：

| 功能 | 实现方式 |
|------|---------|
| **状态持久化** | 存入 `localStorage`，刷新页面不丢失 |
| **跟随系统** | 监听 `prefers-color-scheme` 自动匹配 |
| **平滑过渡** | CSS `transition: all 0.35s ease` |
| **全局控制** | `window.toggleTheme()` 一键切换 |

#### 2.3.3 设计亮点：太阳/月亮主题按钮

我们在导航栏放置了一个**动画主题切换按钮**，设计如下：

- **视觉设计**：椭圆形按钮，内含太阳/月亮轨道动画
- **白天模式**：蓝色渐变背景 + 旋转光芒 + 浮动云朵
- **夜间模式**：深色渐变背景 + 闪烁星星 + 静谧月亮
- **交互反馈**：悬停放大、点击压缩、状态渐变

这一设计不仅是装饰，更体现了前端动画与 CSS 技术的结合。

### 2.4 滚动动画系统 - IntersectionObserver

#### 2.4.1 传统方案的问题

早期实现滚动动画通常监听 `window.scroll` 事件：

```javascript
window.addEventListener('scroll', () => {
  if (window.scrollY > 100) {
    element.classList.add('in');
  }
});
```

**问题**：滚动事件触发频率极高（每帧可能触发），造成性能浪费。

#### 2.4.2 我们的方案：IntersectionObserver

`IntersectionObserver` 是浏览器原生 API，专为「元素进入视口」场景优化：

```javascript
const animObs = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.classList.add('in');  // 添加动画类
    }
  });
}, { threshold: 0.08 });  // 元素 8% 可见时触发
```

**优势**：
- 异步执行，不阻塞主线程
- 性能消耗极低
- 代码简洁

#### 2.4.3 动画实现细节

| 类名 | 含义 | 动画效果 |
|------|------|---------|
| `.an` | 待触发元素 | 初始 `opacity:0; transform:translateY(40px)` |
| `.in` | 已触发元素 | 渐变至 `opacity:1; transform:none` |
| `.d1` ~ `.d6` | 延迟阶梯 | `transition-delay: 0.08s` ~ `0.5s` |

效果：元素依次错落进入视口，产生「瀑布流」般的视觉体验。

### 2.5 国际化（i18n）

#### 2.5.1 实现方案

使用两个全局变量控制语言切换：

```javascript
let lang = 'zh';  // 当前语言

const T = {
  zh: { heroTitle: '因开放而精彩', ... },
  en: { heroTitle: 'Open for Brilliance', ... }
};

function applyLang() {
  const dict = T[lang];
  // 遍历所有需要翻译的 id，替换文本
  document.getElementById('heroTitle').textContent = dict.heroTitle;
  // ...
}
```

#### 2.5.2 应用场景

- 导航栏链接文本
- Hero 区域标题与副标题
- 卡片内容与描述
- 按钮标签与提示文字
- 表单字段与错误提示

**切换方式**：导航栏右侧语言切换按钮（EN/中）

### 2.7 本地数据存储 - localStorage

#### 2.7.1 为什么需要「数据库」？

网站有「加入我们」申请功能，需要收集用户姓名、邮箱、意向部门、申请理由。

**传统方案**：使用 PHP/Python 等后端语言 + MySQL 数据库

**我们方案**：使用浏览器 `localStorage` 模拟数据库

```javascript
// db.js 中的核心方法
function addApplication(appData) {
  const apps = getAllApplications();
  apps.unshift({ id: Date.now(), ...appData, createdAt: Date.now() });
  localStorage.setItem('cduestc_applications', JSON.stringify(apps));
}
```

#### 2.7.2 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Number | 时间戳作为唯一 ID |
| `name` | String | 申请人姓名 |
| `email` | String | 邮箱地址 |
| `dept` | String | 意向部门 |
| `message` | String | 申请理由 |
| `createdAt` | Number | 创建时间戳 |

#### 2.7.3 管理后台功能

管理员登录后可：
- 查看所有申请记录
- 按部门筛选
- 按邮箱搜索
- 删除单条记录
- 导出为 CSV 文件

### 2.8 响应式设计

#### 2.8.1 断点策略

| 断点 | 屏幕宽度 | 布局调整 |
|------|---------|---------|
| 默认 | >1024px | 5列成员网格、3列项目网格 |
| 平板 | 768px-1024px | 3列成员、2列卡片 |
| 移动 | <768px | 汉堡菜单、1列布局 |
| 小屏 | <480px | 紧凑间距、居中文字 |

#### 2.8.2 移动端适配

- **汉堡菜单**：768px 以下显示，点击展开侧边栏
- **触摸滑动**：轮播图支持左右滑动切换
- **弹性布局**：使用 `flex` 和 `grid` 代替固定像素
- **字体缩放**：`clamp()` 函数实现响应式字号

```css
.hero h1 {
  font-size: clamp(38px, 6.5vw, 76px);  /* 最小38px，最大76px，中间自适应 */
}
```

## 七、团队技术挑战与解决

### 7.1 挑战一：主题切换的「闪屏」问题

**问题**：页面加载时，如果直接渲染默认主题再切换，会产生短暂闪烁。

**解决**：ThemeManager 在 HTML 解析前即执行，通过内联 script 尽早设置 `data-theme` 属性：

```html
<script src="./js/theme.js"></script>
<!-- 脚本在 <body> 开头就执行，此时 DOM 还未渲染 -->
```

### 7.2 挑战二：移动端导航的「层叠上下文」

**问题**：汉堡菜单展开时，可能被页面其他元素遮挡。

**解决**：给导航栏设置足够大的 `z-index: 1000`，并使用 `position: fixed` 确保脱离文档流。

### 7.3 挑战三：localStorage 的「同源策略」限制

**问题**：localStorage 只能被同源（同协议+域名+端口）页面访问。

**解决**：所有页面均使用相对路径，保持在同一源下。
